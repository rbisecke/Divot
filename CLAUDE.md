# Divot — on-device golf swing analysis (SwiftUI + SwingCore)

Personal iOS app. SwiftUI over a validated Swift package (`SwingCore`) that uses Apple
Vision + AVFoundation. All analysis runs on-device; video never leaves the phone.
Not distributed on the App Store.

## Hard rules (never violate)
- **Privacy:** never commit real swing video or personal session data. Test media must be
  synthetic (`ios/DivotTests/Fixtures/placeholder_swing.mov`, generated by
  `tools/make_placeholder_clip.swift`). The recorded `sample_swing.pose.json` is skeleton
  coordinates only (no imagery) and is the sole recorded input. Real clips are gitignored.
- **Signing:** never commit Team IDs, bundle-signing identities, provisioning profiles, certs,
  or `*.p8` keys. The real `DEVELOPMENT_TEAM` lives only in `ios/Config/Signing.local.xcconfig`
  (gitignored). `ios/Config/Signing.xcconfig` (committed) `#include?`s it.
- **Commits:** no AI/Claude attribution or `Co-Authored-By` trailers. No em dashes in prose.
- **Never `cd` to run a tool.** Use its directory flag: `git -C`, `make -C`,
  `swift --package-path`, `xcodegen --spec/--project`, `xcodebuild -project`.

## Architecture
- `SwingCore/` — pure engine, no app imports. Pose estimation, event detection, metrics,
  club-aware `FaultEvaluator`, bundled pro `ReferenceStore`, `Trends`, and the open
  `ClubSpec`/`Bag` club model. Analysis is category-based; new club categories fold onto the
  four benchmark/reference families via `analysisFamily`. Bundles `Resources/reference` (8 pro
  templates). Headless self-check: `SwingCoreCheck`.
- `ios/` — SwiftUI app over `SwingCore` (declared as a local package, `path: ../SwingCore`).
  A `PoseProvider` seam (Vision real / Replay mock) lets the full pipeline run device-free on
  the Simulator against `sample_swing.pose.json`.

## Build & validate
- **Engine (T1, headless, no Xcode needed):** `make validate`  → expect `ALL PASS, 111 checks`
- **Generate the Xcode project:** `make generate` (XcodeGen from `ios/project.yml`; the
  `.xcodeproj` is generated, not committed)
- **App (T2, simulator):** `make test` → expect 56 unit (+4 device-skipped) + 2 UI, 0 failures
- **Device (T3, local only):** `make device-test DEVICE=<udid>` (`xcrun devicectl list devices`)

## Validation tiers (gate every change)
- **T1** engine golden checks (`SwingCore/validate.sh`) — regression baseline must stay identical.
- **T2** simulator XCTest: unit + a11y audit + screenshot tour. Run before every commit.
- **T3** device: real Vision pipeline, ball-flight, 240fps, 3D pose, haptics. Needs a real iPhone.
  Only T1 + T2 run in CI (no camera / Vision body-pose model on CI or the Simulator).

## Gotchas
- The Simulator has no body-pose model → live Vision returns nothing; the app/tests use the
  Replay provider (pose JSON). Device-only tests are `#if targetEnvironment(simulator)` skipped.
- The on-device pipeline test needs a real clip at `ios/DivotTests/Fixtures/sample_swing.mov`
  (gitignored); it `XCTSkip`s if absent, so drop your own to run the full device suite.
- The a11y audit enforces structural checks but tolerates `.contrast` — the iOS 26.x Simulator
  contrast audit regressed and flags even ~9-15:1 text (verified against the prior repo on the
  same runtime). Re-enable when the runtime scoring is fixed.
- Regenerate the project (`make generate`) after adding/removing files before building.
